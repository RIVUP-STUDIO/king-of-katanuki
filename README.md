# KING OF KATANUKI（試作）

昔ながらの縁日「型抜き」を再現する、超シンプル集中型のスマホゲームプロトタイプです。

ストーリーもキャラクターもボス戦もありません。
ルールはひとつ。線からズレず、なぞりきる。それだけです。

## 遊び方

1. ステージを選ぶ（日の丸／ハート／星）
2. 画面上を指でなぞる（指先そのものではなく、指から少し離れた場所に「針先」が表示され、そこが判定点になります）
3. 針先が線の上に乗ると **緑**（セーフ）
4. 外側にズレると **白**（まだセーフ、戻せばOK）
5. 内側にズレると **赤**（即ゲームオーバー）
6. 線を100%なぞりきったらクリア

一筆書きの必要はなく、どこからでも好きな順番でなぞれます。
クリアタイムが画面に表示されます（ランキング機能は未実装）。

## ファイル構成

```
king-of-katanuki/
├── index.html   HTML構造
├── style.css    見た目・レイアウト
├── script.js    ゲームロジック（Canvas描画・判定・入力）
└── README.md    このファイル
```

## 動かし方

`index.html` をブラウザで開くだけで動きます。ビルドやサーバーは不要です。
スマホでの動作確認を推奨します（`touchstart`/`touchmove` を使用しているため、PCではマウスドラッグで代用できます）。

```
open index.html
```

またはローカルサーバーで:

```
npx serve .
```

## ゲームの仕組み（概要）

- 型の輪郭は「中心からの角度ごとの半径」として数式で表現しています（`script.js` の `STAGES` / `heartRadius` / `starRadius`）。
  円はもちろん、ハートや星のような複雑な形も同じ判定ロジックで扱えます。
- 輪郭を360分割した「バケツ」に分け、緑判定で通過したバケツを記録して進捗（トレース率）を出しています。
- 指の位置と実際の判定点（針先）をあえてズラすことで、指で判定点が隠れないようにしています。
- 状態が変化した瞬間（緑・白・赤）に振動（Vibration API）を鳴らします。
  - iOS Safari は Vibration API 非対応のため振動しません（ブラウザ側の制限）。
  - Android Chrome 等では動作します。
